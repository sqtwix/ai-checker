# НейроЭксперт (Анализатор результатов тестирования)

Интеллектуальная система для автоматического анализа результатов тестирования студентов с использованием локальных и облачных нейросетевых моделей (LLM). Система выявляет массовые ошибки, оценивает ответы в сравнении с эталонами, обнаруживает аномалии в поведении студентов (например, слишком быстрое прохождение) и предоставляет рекомендации преподавателям.

## Архитектура системы

Система состоит из пяти основных контейнеризированных сервисов:
1. **api-core** (ASP.NET Core 9.0) — ядро бэкенда, отвечает за авторизацию, хранение истории, валидацию файлов и оркестрацию задач.
2. **frontend** (React + Vite) — пользовательский интерфейс для работы преподавателей и просмотра аналитики.
3. **ai-driver** (FastAPI + Python 3.13) — координатор ИИ-агентов. Реализует последовательный конвейер анализа: методический разбор ответов -> поиск аномалий -> сборка финальной статистики и рекомендаций.
4. **qwen-local** (llama.cpp) — локальный сервер инференса для обеспечения работы системы в offline-режиме без внешних API-ключей.
5. **Базы данных** — PostgreSQL (хранение пользователей и отчетов) и Redis (кэширование и распределенная очередь задач).

## Быстрое развертывание

### 1. Требования к окружению
* Установленный **Docker** и **Docker Compose**.
* Локальная модель `Qwen3.5-0.8B-Q8_0.gguf` (или аналогичная), помещенная в папку `models/` в корне проекта для обеспечения работы в offline-режиме.

Файлы локальной модели изначально не лежат в репозитории, так как очень много весят. Команда для скачивания Qwen3.5-0.8B-Q8_0.gguf:
```shell
curl -LO https://huggingface.co/mozilla-ai/llamafile_0.10/resolve/main/Qwen3.5-0.8B-Q8_0.llamafile

# Make it executable (macOS/Linux/BSD)
chmod +x Qwen3.5-0.8B-Q8_0.llamafile
```

### 2. Запуск системы
Запустите сборку и старт всех контейнеров одной командой из корня проекта:
```bash
docker compose up --build -d
```

### 3. Доступные порты и интерфейсы
* **Frontend**: [http://localhost:3000](http://localhost:3000) — веб-интерфейс преподавателя.
* **Core API**: [http://localhost:5000](http://localhost:5000) — API ядра бэкенда.
* **Swagger документация**: [http://localhost:5000/openapi/v1.json](http://localhost:5000/openapi/v1.json) или через Swagger UI по пути `/swagger` (если настроен).
* **AI Driver (FastAPI)**: [http://localhost:8000/docs](http://localhost:8000/docs) — интерактивная документация к сервису ИИ-агентов.
