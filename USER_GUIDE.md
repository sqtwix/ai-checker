# Руководство пользователя — AI-Checker

Данное руководство содержит инструкции по подготовке материалов, загрузке данных и работе с аналитическими отчетами в системе AI-Checker.

---

## 1. Регистрация и Вход в систему

При первом запуске системы необходимо зарегистрировать учетную запись:
1. Перейдите по адресу [http://localhost:3000](http://localhost:3000).
2. Нажмите **"Создать аккаунт"** на панели авторизации.
3. Введите имя пользователя, почту и пароль (минимум 6 символов). Нажмите **"Зарегистрироваться"**.
4. После регистрации выполните вход, используя свои учетные данные.

---

## 2. Подготовка файлов для анализа

Система ожидает два типа файлов: **Эталонный файл** и **Файлы ответов студентов**.

### А. Эталонный файл (Benchmark)
Описывает структуру теста и правильные ответы. Поддерживаемые форматы: **XLSX**, **CSV**, **JSON**.
* **Для XLSX / CSV**: Файл должен обязательно содержать строку с заголовками колонок. Система ищет вопросы как заголовки колонок, а правильные ответы должны располагаться во второй строке (первая строка данных).
  * *Пример структуры таблицы*:
    | Выберите правильный тип данных | Что вернет len([]) |
    | :--- | :--- |
    | list | 0 |
* **Для JSON**: Должен содержать массив объектов с полями `question_id`, `question_text`, `question_type` и `reference_answer`.

### Б. Ответы студентов (User Responses)
Содержат попытки прохождения теста студентами. Поддерживаются **XLSX**, **CSV** или **ZIP-архив**, содержащий эти файлы.
* Каждая строка представляет собой попытку студента.
* Обязательные колонки (заголовки):
  * `"Пользователь"` — уникальный идентификатор или имя студента.
  * `"Дата"` — дата и время отправки попытки.
  * `"Статус"` — статус попытки (например, "Пройден", "Не пройден").
  * `"Баллы"` — набранные баллы (например, "12 / 15").
* Остальные колонки должны содержать ответы студента на вопросы, заголовки которых **строго совпадают** с заголовками вопросов из эталонного файла.

*Совет: Рекомендуется упаковывать файлы ответов нескольких групп в один `.zip` архив для единовременной пакетной загрузки.*

---

## 3. Запуск анализа данных

1. На главном экране в левой панели нажмите кнопку **"Загрузить кейс"** (или воспользуйтесь зоной Drag-and-Drop).
2. Выберите или перетащите:
   * **Эталонный файл** (XLSX, CSV или JSON).
   * **Файл(ы) ответов студентов** (один или несколько XLSX/CSV, либо один ZIP-архив с ними).
3. В выпадающем списке выберите нейросеть для анализа:
   * **Qwen Local (Offline)** — локальная модель (работает быстро, не требует интернет-соединения).
   * **DeepSeek (Online)** — облачный анализ высокой точности.
   * **Sber GigaChat (Online)** — облачный анализ с упором на русский язык.
4. Нажмите **"Запустить ИИ-анализ"**. Статус выполнения будет отображаться на экране.

---

## 4. Интерпретация результатов анализа

После завершения обработки откроется детальный отчет, содержащий следующие вкладки:

### А. Дашборд (Общая сводка)
* **Показатели успеваемости**: Общий процент правильных ответов группы, количество обработанных файлов, общее число выявленных аномалий.
* **Массовые ошибки**: Список вопросов, на которые не ответило более 40% группы. ИИ группирует ответы, описывает общий паттерн ошибки студентов и предлагает методическую причину путаницы.

### Б. Студенты (Аналитика по каждому учащемуся)
* Отображается список всех студентов, присутствовавших в файлах (включая отличников с 100% верных ответов).
* При клике на студента открывается детальная карточка:
  * Список всех ответов студента с оценкой совпадения с эталоном от ИИ (0–100%).
  * Пояснение ошибки от ИИ-агента для каждого неверного ответа.
  * Персональные поведенческие аномалии (при наличии).

### В. Аномалии поведения
Выявляются автоматически с помощью ИИ-агента:
* **SpeedCheating (Списывание)** — тест пройден аномально быстро при высоком итоговом балле.
* **ExtremeStruggling (Трудности)** — студент потратил экстремально много времени, но показал низкий результат.
* **SuspiciousMatch (Подозрительное совпадение)** — ответы студента имеют подозрительное сходство с эталонным решением или ответами других участников.

### Г. Рекомендации методисту
Список приоритетных действий для улучшения качества курса (например: обновить формулировку конкретного сложного вопроса, добавить теоретическую справку по теме).

---

## 5. Работа с историей и архивом

* **Переименование отчетов**: Вы можете изменить название курса/отчета в истории с помощью иконки редактирования (карандаш) рядом с заголовком.
* **Архивирование**: Чтобы скрыть неактуальный отчет из основного списка истории, нажмите иконку **"Архивировать"** (коробка). Отчет переместится в архив и перестанет перегружать главный экран.
* **Работа с архивом**: Для просмотра архивных отчетов или их восстановления воспользуйтесь фильтрами архива (доступны на фронтенде).

---

## 6. Настройки доступности и UI

В панели настроек (кнопка в левом нижнем углу) вы можете включить:
* **Режим для слабовидящих** (контрастная схема, изменение шрифтов).
* **Минималистичный интерфейс** (скрытие декоративных подсказок для быстрой профессиональной работы).
