# Конкретное ТЗ ИИ агентов анализа ответов

## Общая задача
Создать систему, которая на вход получает правильные ответы нв вопросы и набор файлов с реальными ответами пользователей. Далее ИИ-агенты **анализируют** ответы, **выевляют** правильные/неправильные, **находят** паттерны и тд. Система предоставляет *отчёт*, который можно сохранить в нескольких формах.

**Основная задача ИИ агентов/группы агентов:**
- автоматическая проверка ответов: сравнение ответов пользователя с усредненными данными;
- выявление критических изменений: определение массовых неправильных ответов в конкретных тестовых вопросах;
- анализ временных метрик: отслеживание времени выполнения заданий, выявление аномалий (слишком быстрое/медленное выполнение);
- формирование отчета: подготовка структурированного отчета с выделением критических изменений и рекомендациями по дальнейшим действиям.

## Данные

### Входные (JSON, CSV, папка с файлами для ответов пользователей):
- эталонные ответы
- набор реальных ответов

### Выходные (JSON, excel с подсветкой):
- файл

## Требования

1. Использование как миниму **2 агентов** (РФ и иностранный)
2. Бюджет агента не более 10000 в год
3. Максимальная **модульность и независимость** модулей, чтобы смена агентов занимала как можно меньше времени
4. максимальная простота развертывания - 1 команда
5. Модуль валидации и обработки ошибок (Критерий "Функциональность" - 3 балла):Система должна переваривать любые некорректные данные. Если методист загрузит пустой файл или перепутает колонки местами, система не должна падать с 500 Error. ИИ-агент или валидатор в api-core должен выдать понятное сообщение: "В файле 'answers.csv' отсутствует колонка 'Время прохождения'. Пожалуйста, исправьте".
6. Агентская логика (Критерий "Качество анализа" - 3 балла):Не отправляй всё одним огромным промптом. Раздели ИИ на роли в ai-driver:  Агент-Математик/Статистик: Считает медиану времени, выявляет аномально быстрые прохождения (читтинг/копирование). 
7. Экономика (Бюджет):DeepSeek API невероятно дешев ($0.14 за 1 млн входных токенов на момент начала 2026 года), так что в 10 000 руб. на лимиты  вы уложитесь с огромным запасом, даже выполняя миллионы запросов. Для GigaChat заложите интеграцию через их стандартный Cloud API.  

## Модули системы:
1. **frontend** - 

2. **api-core** - модуль связи фронта и ai-driver.
Стек: .NET (8/9); ASP.NET core + всякие либы
Функционал:
- валидация входных данных;
- сжатие и конвертация входных данных в понятный ai-driver **md/json file**;
- получение результата и преобразование его в нужный формат отчета.

3. **ai-driver** - моудль общения с агентом(api иишки).
Стек: Python (3.12 и выше), fast api, либы для ИИ.
Функционал:
- получение данных от api-core  
- отправление структурированных данных модели (+ системный промт)
- возврат api core результата
- обработка ошибок от апи неройнки 

## Контракты
1. Загрузка эталонных данных и ответов пользователей для запуска асинхронного анализа ИИ-агентами (frontend -> api-core). 
Тип запроса (Content-Type): multipart/form-data   
поле|тип|обязательность|описание

- benchmarkFile - File - обязательно - эталонный файл (csv\json)
- userResponseFiles - File[] (массив) - обязательно - массив файлов ответов пользователей (json/csv)
- modelType - String - необязательно - модель ИИ (deepseek (по умолчанию)/gigachat) 

**Ответы** 
202 Accepted 
Content-Type: application/json
```json
{
  "task_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "message": "Файлы успешно загружены и приняты в обработку."
}
```

400 Bad Request (Ошибка валидации входных данных)
Content-Type: application/json
```json
{
  "error": "Необходимо загрузить хотя бы один файл с ответами пользователей."
}
```
