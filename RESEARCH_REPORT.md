# Отчет об исследовании и обосновании архитектурных решений 

## 1. Архитектура и общая логика проекта 

Система **«НейроЭксперт»** представляет собой интеллектуальную платформу для анализа результатов тестирования студентов. Проект разделен на три ключевых компонента:
1. **Frontend (React/Vite)** — пользовательский интерфейс для загрузки данных, просмотра отчетов, детального анализа ошибок студентов, управления архивом и визуализации статистики.
2. **ApiCore (.NET 9 / ASP.NET Core)** — центральный бэкенд, управляющий аутентификацией, базой данных PostgreSQL, хранением отчетов, импортом/экспортом и оркестрацией задач.
3. **AiDriver (Python / FastAPI)** — микросервис искусственного интеллекта, управляющий взаимодействием с LLM-моделями через мультиагентную систему.

### Ключевые функции и логика работы
*   **Мультиагентный конвейер**: Процесс анализа разделен на специализированных агентов. Вместо одного длинного промпта система использует цепочку:
    1.  *Агент разбора ответов* (выделяет суть ответа студента и сопоставляет с эталоном).
    2.  *Агент классификации ошибок* (группирует ошибки по темам и типам, определяет серьезность).
    3.  *Агент генерации рекомендаций* (составляет персонализированные советы для преподавателя и студента).
*   **Логика коррекции пропущенных студентов (статистический фикс)**:
    Поскольку ИИ-агенты оптимизированы для поиска и разбора *ошибок*, они склонны игнорировать студентов, ответивших на 100% правильно. Из-за этого идеальные результаты пропадали из итоговой статистики. В `AiDriver` внедрен алгоритм пост-обработки `_enrich_and_complete_response`, который сопоставляет результаты со списком участников теста и автоматически добавляет отличников с оценкой `100.0` и комментарием «Ошибок не обнаружено».
*   **Архивация отчетов (Archive Feature)**:
    Реализована концепция сохранения истории без удаления. Отчеты могут быть помечены как архивные (`is_archived = true`). Это исключает их из основного списка, но сохраняет в базе данных. Пользователь может в любой момент переключить фильтр в истории, просмотреть архив и восстановить нужный отчет.
*   **Поддержка архивов ZIP и XLSX**:
    Изначально система поддерживала только файлы CSV и JSON. Для реальных условий хакатона был разработан модуль автоматической распаковки ZIP-архивов и парсинга XLSX-таблиц на стороне сервера, что позволяет загружать исходные выгрузки LMS напрямую.

## 2. Сравнительный анализ ИИ-агентов (Comparing Agents)

В системе заложена поддержка трех ключевых конфигураций ИИ-агентов. Ниже приведена сравнительная таблица их характеристик и применимости:

| Критерий | Локальный Qwen (Qwen-2.5-7B-Instruct) | DeepSeek (V3 / R1) | Sber GigaChat (SberGPT) |
| :--- | :--- | :--- | :--- |
| **Тип развертывания** | Локальный (On-Premise / Docker) | Облачный API | Облачный API (РФ) |
| **Конфиденциальность** | **Максимальная** (данные не покидают сервер) | Средняя (передача по API за рубеж) | Высокая (соответствие ФЗ РФ о персональных данных) |
| **Следование JSON-схеме**| Отличное (с использованием Grammars) | Отличное | Среднее (требует строгой постобработки) |
| **Понимание русского языка**| Высокое | Очень высокое (особенно R1) | Отличное (нативный русский язык) |
| **Стоимость эксплуатации**| Бесплатно (оплата только за железо) | Оплата за токены (низкая) | Оплата за токены |
| **Скорость инференса** | Высокая (при наличии GPU) | Средняя (зависит от нагрузки на API) | Высокая |
| **Автономность** | **Полная** (работает без интернета) | Зависит от внешнего подключения | Зависит от внешнего подключения |

## 3. Технические характеристики моделей 

### 1. Локальный Qwen (Qwen-2.5-7B-Instruct / Qwen-1.5-7B)
*   **Размер контекста**: до 32 768 токенов (в текущей конфигурации ограничено до 8 192 для экономии RAM/VRAM).
*   **Рекомендуемые системные требования**:
    *   *GPU*: NVIDIA RTX 3060/4060 с 12 ГБ VRAM (для квантованного инференса Q4_K_M / Q8_0).
    *   *CPU*: 6+ ядер, не менее 16 ГБ RAM (при работе без видеокарты скорость снизится до 2-5 токенов/сек).
*   **Формат квантования**: GGUF (используется движок `llama.cpp` внутри контейнера `qwen-local`).
*   **Особенности**: Отличается высокой точностью в структурировании данных и генерации строго валидного JSON.

### 2. DeepSeek (V3 / R1)
*   **Размер контекста**: 64k - 128k токенов.
*   **Требования**: Не требует локальных видеокарт, работает по протоколу OpenAI API.
*   **Особенности**: Модель R1 использует встроенное пошаговое рассуждение (Chain of Thought), что делает её незаменимой для поиска неочевидных логических ошибок в программном коде или сложных текстовых ответах гуманитарного профиля.

### 3. Sber GigaChat
*   **Размер контекста**: до 32 000 токенов.
*   **Требования**: Требует авторизации через Сбер ID / Бизнес-портал, обмен данными по шифрованному каналу TLS с сертификатами Минцифры.
*   **Особенности**: Идеально подходит для государственных и образовательных учреждений с жесткими требованиями к импортозамещению и локализации трафика внутри РФ.

## 4. Обоснование выбора текущей конфигурации

По умолчанию система настроена на работу с **локальным Qwen** (`qwen_local`). Данный выбор обусловлен следующими факторами:

1.  **Независимость и безопасность**: В образовательных учреждениях обрабатываются персональные данные студентов (ФИО, оценки, академическая успеваемость). Передача этих данных на внешние зарубежные серверы (такие как OpenAI или зарубежные шлюзы DeepSeek) может нарушать требования законодательства (например, ФЗ-152). Локальный запуск гарантирует 100% конфиденциальность.
2.  **Экономическая эффективность**: Система анализа результатов тестов может обрабатывать сотни тысяч студенческих ответов в период сессий. Использование платных API привело бы к постоянным расходам бюджета, в то время как локальная видеокарта окупает себя за несколько недель активного использования.
3.  **Стабильность**: Работа системы не зависит от наличия интернет-соединения, санкционных блокировок, изменения тарифов или лимитов на запросы (Rate Limits) со стороны внешних провайдеров.
4.  **Качество работы**: Семейство моделей Qwen-2.5 является мировым лидером среди моделей с открытым исходным кодом (open-source) в категории до 7-14 млрд параметров, превосходя по качеству работы с кодом и структурированным выводом многие закрытые коммерческие аналоги.
